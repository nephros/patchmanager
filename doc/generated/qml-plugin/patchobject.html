Detailed Description

An element to be used as content of an PatchManagerModel

Upon construction, the data parameter is mapped to PatchObject::detail

Member Function Documentation

PatchObject::PatchObject(const QVariantMap &data, QObject *parent = nullptr)

Default constructs an instance of PatchObject.

[virtual] PatchObject::~PatchObject()

Destroys the instance of PatchObject. The destructor is virtual.

[slot] void PatchObject::apply(QJSValue callback = QJSValue::UndefinedValue)

Calls PatchManager::applyPatch with the patch name. Calls callback afterwards (if callable).

Note: Does nothing if the patched property is true.

bool PatchObject::busy() const

Returns true when an internal operation is in progress.

Note: Getter function for property busy.

[slot] void PatchObject::resetState()

Calls PatchManager::resetState with the patch name.

void PatchObject::setData(const QVariantMap &data)

Fills the PatchObject's properties from data.

If there is a display_name field in data, and data is treated as regular patch metadata (isNewPatch is set to true). Otherwise, the field name is used for display name, and the data is treated as legacy metadata.

[signal] void PatchObject::toBeDestroyed(PatchObject *object)

This signal is emitted when object is about to be destroyed. (Duh.)

[slot] void PatchObject::unapply(QJSValue callback = QJSValue::UndefinedValue)

Calls PatchManager::unapplyPatch with the patch name. Calls callback afterwards (if callable).

Note: Does nothing if the patched property is false.

[slot] void PatchObject::uninstall()

Calls PatchManager::uninstallPatch with the patch name.
